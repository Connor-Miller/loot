loot — storage foundation bake-off

How we chose the foundational storage model, and the evidence behind it.

Verdict: the encrypted content-addressed DAG is loot's foundation. decided

Not a judgment call — the deciding test produced hard numbers. Under loot's per-content encryption, the CRDT silently dropped concurrent edits (0 of 4 survived); the DAG surfaced conflicts safely and was ~4.5× faster at scale. Full record below. Recorded in docs/adr/0002-encrypted-dag-foundation.md.

The question

loot's thesis: visibility and permissions belong to content and changes, not the repository. You can commit a .env (encrypted, only keyholders decrypt), keep files private inside a shared repo, and embargo a security fix until a reveal time. Every one of those reduces to the same primitive: per-unit encryption with a key-access policy.

Two storage models could host that primitive. We refused to pick on intuition and instead built both as competing spikes, then measured.

DAG
Encrypted content-addressed DAG. Git-like objects in a DAG, but each object encrypted independently; addressed by ciphertext hash, with a separate plaintext identity hash for dedup. Log-structured store (no loose-file explosion), in-memory capable.
CRDT
Automerge document store. The repo is a CRDT document; the working tree is a projection. Native strength: conflict-free convergence of concurrent edits.

Methodology

The bake-off was designed so the comparison could not be rigged:

  1. One shared contract. Both models implement the same Repo trait in loot-core (put/get/commit/checkout + sync via bundle/apply). Neither could change the contract.
  2. One shared workload. All scenarios live in loot-bench and are generic over R: Repo — identical code scores both spikes. Neither defines its own benchmark.
  3. Sync was put into the bake-off on purpose. The easy mistake was to measure only local performance, which would bias toward the DAG before any code ran. We added concurrent-offline-edit convergence precisely so the CRDT could show its one structural advantage.
  4. Convergence rule fixed up front (ADR 0001): per-content, decrypt-then-merge. Keyholders merge; a peer lacking the key may only relay ciphertext. Both models obey the same rule.
  5. We measured the real artifact, not the summary. When the CRDT reported "0 conflicts," we did not accept it. We materialized the converged file to disk and counted how many peers' edits actually survived. That single check flipped a false win into the deciding evidence.
  6. Reproducible. Every number below comes from cargo test --release over the two spike crates, run repeatedly to confirm stability.

Axis 3 — Sync (the deciding test)

The CRDT's best case: 4 keyholders edit the same file concurrently while offline, then converge. This is the only workload where a true CRDT should beat a 3-way-merge DAG. We ran it and read back the converged file.

ModelConflictsEdits survivingOutcome
DAG31 of 4 Safe. Peer 0's edit auto-applied; the other 3 flagged for a human — nothing lost.
CRDT00 of 4 Silent data loss. "Conflict-free" meant every peer's edit was overwritten.
Why the CRDT lost its own best case. Content is stored encrypted. Automerge merges by reading content — but it cannot read ciphertext, so it treats each file as an opaque register and resolves concurrent writes last-writer-wins. The CRDT's one advantage is structurally unavailable exactly where loot needs it. In a non-encrypted VCS, Automerge would genuinely shine here; that is not the product we are building.

For source control, a conflict a human resolves is strictly better than an edit that silently vanishes. The DAG fails safe; the CRDT fails silent.

Decision & honesty

We lock the encrypted content-addressed DAG. The decision is backed by the numbers above, not by preference. To stay honest, the DAG has its own sharp edge we are not hiding: dedup keys on a plaintext identity hash, which leaks a same-plaintext equality oracle and partially undercuts the privacy thesis. It's a tracked follow-up (drop cross-key dedup, or use a keyed identity hash) — a fixable detail, not a foundational defect like the CRDT's silent loss.
