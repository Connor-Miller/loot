# Dogfood pilot: one loot-only working day (2026-07-07)

Resolves wayfinder ticket [#56](https://github.com/Connor-Miller/loot/issues/56)
(map [#54](https://github.com/Connor-Miller/loot/issues/54)). A scratch copy of
this repo (80 files, ~900 KB) driven through a full working day on Windows:
init → `.lootattributes` → snapshot/describe/new cycles → docks → local relay →
second identity (bob) → concurrent edits → conflict resolution. Every gap
recorded, tagged **blocking / annoying / cosmetic**; security-grade findings
called out on top.

## Security-grade findings

1. **BLOCKING — `.lootattributes` forward-slash globs fail open to Public on
   Windows.** `docs/private/* restricted=connor` matched nothing (snapshot
   paths are OS-native, `docs\private\secrets.md`); the secret was sealed
   **Public** with no warning. `docs\private\*` (backslash) matches; exact
   paths are separator-normalized; verb *arguments* (`loot grant docs/private/…`)
   normalize fine — the bug is isolated to attribute glob matching. The one
   portable spelling a git user would write silently discloses. Fail-open is
   the worst failure mode for a privacy-first VCS. → issue filed.

2. **BLOCKING — editing `.lootattributes` silently demotes restricted content
   to Public.** Removing a path's line (a typo, a refactor of the file)
   re-seals that path Public on the next snapshot — no warning, no `migrate`
   ceremony, even though CONTEXT.md declares visibility migration a
   first-class operation. In the pilot, `secret.txt` (previously
   `restricted=connor`) went Public this way, was pushed, and **bob read it**.
   Related: `.lootattributes` itself is excluded from snapshots, so the policy
   never travels to peers or survives a re-clone. → issue filed.

3. **BLOCKING (design) — no remedy for a mis-sealed secret.** Once a change is
   finalized, plaintext sealed under an ANYONE key is immutable history;
   `migrate`/`maroon` only protect the future. git has BFG/filter-branch;
   loot has nothing. Given findings 1–2 make accidental public seals *likely*,
   a purge/rewrite story (or a pre-finalize visibility gate) is needed.
   → issue filed.

## Blocking for daily driving

4. **No ignore mechanism.** A `target/` of 301 files / 38 MB was sealed into
   the working change by one `status` (`.loot` grew 42 MB); finalizing would
   make it immutable, travel on every push, and record 301 deletions when
   removed. Snapshot speed is fine (0.7 s) — the cost is history pollution.
   → `.lootignore` issue filed.

5. **No diff, anywhere** (existing [#1](https://github.com/Connor-Miller/loot/issues/1),
   confirmed). You cannot see what a change touched, what your working change
   contains, or what a conflict's two sides say
   ([#13](https://github.com/Connor-Miller/loot/issues/13)). The unexplained
   CONTEXT.md conflict (finding 6) was uninvestigatable from inside the tool.

6. **Spurious conflict on content neither side edited.** After bob's clone
   re-sealed every public path under fresh keys, connor's pull reported
   `CONTEXT.md conflict (needs resolution)` — a file neither identity had
   touched since the common base. Decrypt-then-merge should classify identical
   plaintext as *Converged*. Erodes trust in the converge path (the crown
   jewel). Repro steps in the issue. → issue filed.

7. **`loot gc` has vanished.** Implemented and merged
   ([#17](https://github.com/Connor-Miller/loot/issues/17), commit `e1d227e`),
   absent from today's CLI — likely lost in merge `2baaf54`. With finding 4
   producing orphaned objects, gc matters. → regression issue filed.

## Annoying

8. **`status` dumps the full tree** (82 lines) with no changed/unchanged
   distinction (existing [#7](https://github.com/Connor-Miller/loot/issues/7),
   confirmed). You cannot answer "what did I just do".
9. **`pull` prints a verdict for every path** — the 2 real conflicts were
   buried in 80 lines of `merged`. Also: identical/untouched paths report
   `merged` where ADR 0001's taxonomy says *Converged* — the interesting
   verdict is indistinguishable from the boring one. (Evidence posted to CA3
   [#51](https://github.com/Connor-Miller/loot/issues/51).)
10. **`pull` doesn't auto-surface** (existing
    [#3](https://github.com/Connor-Miller/loot/issues/3), confirmed) — the
    tree is stale until you remember `loot surface`.
11. **Unknown flags are silently ignored.** `loot log --path README.md`
    printed the full log as if the filter worked. Worse than erroring.
    → issue filed.
12. **`pulled 82 change(s)`** — the count is objects/paths, not changes (~5
    existed). Misleading unit.
13. **Multi-head log is confusing.** A dock tip counts as "diverged" (`3 heads
    — diverged; run loot apply to converge`), and the advice names the wrong
    verb post-network (`pull`; and dock convergence is CA2's `dock merge`,
    which doesn't exist yet).
14. **No per-command help.** `loot status --help` *runs status* (or errors);
    only the global usage block exists.

## Cosmetic

15. `loot docks` shows tip ids but not descriptions.
16. `log` shows `[logged by 13d54d46…]` for unknown peers — correct by design
    (names are local) but could hint `loot peer add`.

## What worked (genuinely good)

- **The thesis, when the seal is right**: bob's clone materialized everything
  *except* `docs/private/` — silently, correctly; his re-snapshot carried the
  sealed path forward unchanged. The core promise held.
- **init UX**: keypair + identity + actionable tips in one step.
- **`status -m`**: snapshot+describe in one verb kills the git add/commit
  ceremony as intended.
- **Docks**: create/switch/edit/switch-back round-trip flawless; nothing lost.
- **Speed**: snapshot 80 files 0.13 s, 380 files 0.69 s; push 756 KB 0.23 s;
  clone instant. No performance gaps at this scale.
- **Honest messaging**: push says "this published your sealed content";
  `resolve` walks you to `loot new`. The voice is right.
- **Conflict resolution flow**: `conflicts` → compose file → `resolve` →
  finalize worked first try.

## Verdict for triage ([#55](https://github.com/Connor-Miller/loot/issues/55))

The ritual is *almost* livable — docks, snapshot ceremony, and sync mechanics
are already better than expected. But daily driving is gated on: the two
attributes security bugs, an ignore mechanism, a diff verb, and delta-only
status/pull output. The DX tail items confirmed by evidence: #1, #3, #7, #13.
Not yet needed at this scale: S5 wants-negotiation, S6 resumable transfer,
compression tuning, #26 parallel writes.
